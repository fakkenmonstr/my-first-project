# РЭА Агент — Документация

## 1. Что это?

Веб-приложение с AI-агентом, который ищет актуальные темы и тренды в области радиоэлектронной аппаратуры по заданным пользователем направлениям, используя поиск в интернете в реальном времени.

---

## 2. Целевая аудитория

**Инженер-конструктор РЭА** — специалист, которому нужно быть в курсе актуальных разработок, новых компонентов, стандартов и технологий в своей области. Приложение помогает быстро находить релевантную информацию без ручного обхода множества источников.

Приложение рассчитано на **десктоп** (браузер на ПК), мобильная адаптация не является приоритетом.

---

## 3. Какую проблему решает?

Инженер тратит время на ручной поиск по множеству ресурсов (IEEE, Habr, EEVblog, производители компонентов и т.д.), чтобы найти актуальные темы для разработки. Приложение автоматизирует этот процесс: пользователь задаёт интересующие направления — агент сам ищет, анализирует и структурирует информацию.

---

## 4. Основные функции

| Функция | Описание |
|---|---|
| Поиск по направлениям | Пользователь задаёт до 10 тегов-направлений (например: SDR, LoRa, силовая электроника) |
| Веб-поиск в реальном времени | Агент ищет в интернете через Claude AI с инструментом web_search |
| Стриминг результатов | Ответ отображается по мере генерации, без ожидания |
| Структурированный вывод | Результаты в формате Markdown: заголовки по темам, описание, ссылки на источники |
| Сохранение интересов | Список тегов сохраняется в браузере (localStorage) |
| Экспорт результатов | Копирование в буфер обмена или скачивание в виде `.txt` файла |
| Оценка результата | 👍 / 👎 — оценки сохраняются и влияют на системный промпт при следующих поисках |
| База знаний | Сохранение отдельных результатов; агент получает краткое саммари базы в контексте |

---

## 5. Хранение данных

| Данные | Где хранится |
|---|---|
| Теги-интересы | localStorage браузера |
| База знаний (записи) | SQLite файл на сервере (`knowledge.db`) |
| Оценки результатов | SQLite файл на сервере (`knowledge.db`) |
| API-ключ Anthropic | `.env` файл на сервере, не попадает в браузер и не коммитится в git |

**Лимиты:**
- Максимум 10 тегов-интересов одновременно
- Максимум 200 записей в базе знаний (старые вытесняются при превышении)
- В контекст агента передаётся не вся база, а саммари последних 20 записей, чтобы не переполнять контекст

---

## 6. Обработка ошибок

| Ситуация | Поведение |
|---|---|
| Claude API недоступен | Сообщение об ошибке в области результатов, кнопка «Попробовать снова» |
| Нет интернета | Сообщение «Нет соединения» |
| Поиск занял более 60 сек | Таймаут с сообщением об ошибке |
| Пользователь отменил поиск | Кнопка «Стоп» прерывает стриминг |
| База знаний пуста | Заглушка с подсказкой «Сохраните первый результат поиска» |
| Поиск ничего не нашёл | Сообщение «Ничего не найдено — попробуйте изменить запрос» |

---

## 7. Как работает оценка результатов

- После завершения поиска появляются кнопки 👍 / 👎
- Оценка сохраняется в БД вместе с тегами, которые использовались при поиске
- При следующем поиске агент получает в системном промпте: «Пользователю нравится: [темы с 👍]. Пользователю не нравится: [темы с 👎]»
- Это корректирует стиль и направление поиска, но не гарантирует точного совпадения

---

## 8. Как работает база знаний

- Пользователь нажимает «Сохранить в базу» после завершения поиска
- Запись содержит: дата, теги поиска, полный текст результата
- При следующем поиске агент получает саммари последних 20 записей базы знаний
- Это позволяет агенту не повторять уже известное и учитывать накопленный контекст
- Записи можно удалять вручную на странице `/knowledge`

---

## 9. Страницы / экраны

### Главная страница (`/`)

- **Заголовок** — название приложения
- **Блок интересов** — теги-направления (добавить / удалить, макс. 10). Примеры: `SDR`, `LoRa`, `FPGA`, `Силовая электроника`, `Беспроводные сети`
- **Поле запроса** — необязательное текстовое уточнение
- **Кнопка «Найти»** — запускает агента; во время поиска меняется на «Стоп»
- **Индикатор поиска** — показывает что агент сейчас делает (ищет / анализирует / пишет ответ)
- **Область результатов** — стриминг Markdown-текста с темами и ссылками
- **Панель действий** (появляется после завершения): кнопки 👍 / 👎, «Скопировать», «Скачать .txt», «Сохранить в базу»

### База знаний (`/knowledge`)

- **Список записей** — карточки: дата, теги, первые 2 строки текста
- **Поиск по базе** — фильтрация по тексту или тегу
- **Удаление записи** — кнопка на каждой карточке с подтверждением
- **Пустое состояние** — заглушка с подсказкой
